# Research system

This folder tracks validation by stages. The idea must become more precise, better sourced, and less hypothetical at each step.

## Validation ladder

1. `L0 - Problem hypothesis`
   You have a pain-point thesis, a target user, and a narrow wedge.
2. `L1 - Desk evidence`
   You have recent public evidence that the pain exists, is urgent enough, and is safe enough to keep validating.
3. `L2 - Validation Gate (Mom Test OR Market Gravity Test)`
   - **B2B / Accessible Audiences:** 3-5 expert calls confirming buyer reality, integration, and compliance.
   - **B2C / Indie Devs (Hard to reach):** Market Gravity Test (Scorecard + Evidence Matrix + Skin in the game metrics via Waitlist/V0).
   - **Adversarial Fallback:** Strict Mom Test Simulation Bypass (Goal: try to INVALIDATE the idea).
4. `L3 - Pilot-ready offer`
   You have one narrow offer, one measurable outcome, and a realistic approval path (or high-conversion waitlist).
5. `L4 - Willingness-to-pay proof`
   You have a paid pilot, signed LOI, $100+ in pre-sales, or undeniable user traction.

## Files

- `evidence-matrix.csv`: dated claims and sources
- `scorecard.md`: go/no-go scoring against fixed axes
- `open-questions.md`: what still blocks the next stage

## Private data rule

Do not commit raw interviews, contact lists, or private customer notes.
Keep those only in ignored paths such as:

- `research/private/`
- `research/interviews/`
- `research/raw/`
- `research/contact-lists/`
